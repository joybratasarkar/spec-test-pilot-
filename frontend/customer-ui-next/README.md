# Customer UI (Next.js)

Browser UI for running the QA agent across multiple domains with one click.

## Features

1. Multi-domain selection (`ecommerce`, `healthcare`, `logistics`, `hr`)
2. Full run trigger via existing `run_qa_domain.sh`
3. Live process logs via SSE stream (`/api/jobs/:jobId/events`)
4. Automatic persistence verification support
5. Interactive report viewer:
- summary cards (pass rate, quality gate, RL steps/buffer, run reward)
- scenario table with pass/fail filter and text search
- raw JSON / Markdown rendering
6. In-UI "Customer API + Agent Flow" panel showing each API and runtime step I/O
7. Generated test scripts explorer:
- list generated files (pytest, jest, curl, restassured)
- preview script source directly in UI
8. Agent R&D panel:
- selection policy details
- top decision table
- reward breakdown + weakest pattern table
9. Customer output sequence:
- `1) Test Scripts Generated By Agent`
- `2) Cases Tested By Agent`
- `3) Final QA Report`

## Run

1. Install dependencies:
```bash
cd frontend/customer-ui-next
npm install
```

2. Start dev server:
```bash
npm run dev
```

3. Open:
```text
http://localhost:3001
```

## Run With FastAPI Backend

Terminal 1 (FastAPI backend):

```bash
./backend/run_customer_backend_fastapi.sh
```

Terminal 2 (Next.js frontend):

```bash
NEXT_PUBLIC_BACKEND_BASE_URL=http://127.0.0.1:8787 npm run dev -- -p 3001
```

## Notes

1. API routes run on Node.js runtime and spawn `backend/run_qa_domain.sh`.
2. Run outputs are stored under `/tmp/qa_ui_next_runs`.
3. Per-domain checkpoints are stored under `/tmp/qa_ui_next_checkpoints`.
4. Frontend receives real-time updates over SSE and falls back to polling if stream disconnects.

## Customer API Contract

If frontend and backend are split (FastAPI backend), the same API contract is used:

1. `POST /api/jobs`
- starts run
- request body:
```json
{
  "domains": ["ecommerce", "healthcare"],
  "tenantId": "customer_default",
  "prompt": "optional context",
  "maxScenarios": 16,
  "passThreshold": 0.7,
  "baseUrl": "http://localhost:8000",
  "customerMode": true,
  "verifyPersistence": true,
  "customerRoot": "~/.spec_test_pilot"
}
```
- response:
```json
{ "jobId": "abc123...", "status": "queued" }
```

2. `GET /api/jobs`
- list all jobs

3. `GET /api/jobs/{jobId}?tail=1200`
- snapshot for one job: status, metadata, logs tail, domain results

4. `GET /api/jobs/{jobId}/events`
- SSE stream for realtime updates
- event types: `snapshot`, `done`

5. `GET /api/jobs/{jobId}/report/{domain}?format=json|md`
- domain report fetch

6. `GET /api/jobs/{jobId}/generated-tests/{domain}`
- list generated test script metadata

7. `GET /api/jobs/{jobId}/generated-tests/{domain}/{kind}`
- fetch generated script content
